# Media Explorer

Media Explorer — мобильное Android-приложение для поиска и просмотра информации о фильмах. Приложение использует API сервиса TMDB (The Movie Database) для получения данных о фильмах, актёрах и трейлерах, а также локальную базу данных Room для хранения пользовательских данных.

Приложение позволяет пользователю просматривать фильмы по различным категориям, выполнять поиск, применять фильтры и сохранять интересующие фильмы в список избранного.

---

# Основные возможности

Приложение предоставляет пользователю следующие функции:

- просмотр популярных фильмов;
- просмотр топ фильмов;
- просмотр трендов;
- просмотр ожидаемых фильмов;
- просмотр общего каталога фильмов;
- поиск фильмов по названию;
- фильтрация и сортировка фильмов;
- просмотр подробной информации о фильме;
- просмотр трейлеров;
- просмотр актёрского состава;
- просмотр страницы актёра;
- добавление фильма в избранное;
- установка статуса просмотра;
- добавление пользовательского рейтинга;
- добавление комментария;
- добавление emoji-реакции.

# Архитектура приложения

Приложение построено на основе архитектурного подхода MVVM (Model-View-ViewModel). Архитектура разделена на несколько уровней, каждый из которых отвечает за свою часть логики приложения.

## UI (User Interface)

Слой пользовательского интерфейса содержит экраны приложения, адаптеры списков и элементы взаимодействия с пользователем.

Основные Activity:

- MainActivity — главный экран приложения;
- DetailsActivity — экран подробной информации о фильме;
- ActorDetailsActivity — экран информации об актёре;
- FavoritesActivity — экран списка избранных фильмов;
- SearchActivity — экран поиска фильмов.

Также используются адаптеры для отображения списков:

- MovieAdapter
- CastAdapter
- FavoritesAdapter
- FilmographyAdapter

---

## ViewModel

ViewModel обеспечивает связь между пользовательским интерфейсом и источниками данных. Этот слой отвечает за подготовку данных для отображения на экране и сохранение состояния интерфейса.

Основные ViewModel:

- MainSharedViewModel
- DetailsViewModel
- ActorViewModel
- FavoritesViewModel

---

## Repository

Repository выполняет роль посредника между источниками данных и ViewModel. Он объединяет работу с сетевым API и локальной базой данных.

Примеры классов:

- FavoritesRepository
- MoviesRepository

Основные задачи Repository:

- загрузка данных из сети;
- сохранение данных в локальной базе;
- предоставление данных ViewModel.

---

## Local Data (Room)

Для хранения пользовательских данных используется локальная база данных Room.

Основные классы:

- AppDatabase — основной класс базы данных;
- FavoriteMovieEntity — таблица избранных фильмов;
- SearchHistoryEntity — таблица истории поиска;
- FavoriteDao — DAO для работы с избранными фильмами;
- SearchHistoryDao — DAO для работы с историей поиска.

В базе данных сохраняются:

- фильмы, добавленные в избранное;
- статус просмотра фильма;
- пользовательский рейтинг;
- комментарии пользователя;
- emoji-реакции;
- история поисковых запросов.

---

## Remote Data (Retrofit)

Для взаимодействия с API TMDB используется библиотека Retrofit.

Основные классы:

- RetrofitClient — настройка сетевого клиента;
- TmdbApiService — интерфейс сетевых запросов;
- DTO-модели для обработки ответов API.

Для выполнения HTTP-запросов используется OkHttp, а для преобразования JSON в Java-объекты используется Gson.

![Экран каталога фильмов](screenshots/catalog.png)

Каталог фильмов поддерживает фильтрацию и сортировку результатов. Пользователь может:
